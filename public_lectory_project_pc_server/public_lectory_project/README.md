# Публичный лекторий — веб-проект

Веб-приложение создано на основе ТЗ для проекта «Публичный лекторий».

## Стек
- Python
- Flask
- SQLite3
- HTML
- CSS
- Waitress
- Gunicorn

## Возможности
- главная страница проекта;
- список лекций;
- список лекторов;
- регистрация слушателей;
- сбор обратной связи;
- простая административная панель для просмотра заявок и отзывов;
- запуск на ПК как сервер для пользователей из локальной сети или через интернет.

## Запуск на ПК как сервер

### Windows
Самый простой способ:
1. Установите Python 3.
2. Откройте папку проекта.
3. Запустите файл:
```text
start_public_server.bat
```
4. После запуска сайт будет доступен на этом ПК по адресу:
```text
http://127.0.0.1:5000
```
5. В локальной сети другие устройства смогут зайти по IP вашего ПК:
```text
http://IP_ВАШЕГО_ПК:5000
```
Пример:
```text
http://192.168.0.15:5000
```

### Linux / macOS
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python run_public.py
```
или:
```bash
./start_public_server.sh
```

## Как узнать IP компьютера
### Windows
```bash
ipconfig
```
Ищите IPv4-адрес, например `192.168.0.15`.

### Linux / macOS
```bash
ip a
```
или
```bash
hostname -I
```

## Доступ для пользователей

### 1. Из локальной сети
Если все пользователи находятся в одной сети Wi‑Fi или через один роутер:
- сервер должен быть запущен на вашем ПК;
- брандмауэр Windows должен разрешать входящие подключения к Python или порту `5000`;
- пользователи открывают `http://IP_ВАШЕГО_ПК:5000`.

### 2. Через интернет из любой точки
Чтобы к сайту могли подключаться пользователи вне вашей сети, нужны дополнительные условия:
- белый внешний IP-адрес **или** DDNS;
- проброс порта на роутере (например, внешний `5000` -> внутренний `5000` на ваш ПК);
- разрешение порта в брандмауэре Windows;
- желательно статический IP внутри локальной сети для вашего ПК;
- для безопасного доступа лучше использовать Nginx/Caddy и HTTPS.

Тогда пользователи будут заходить по внешнему адресу, например:
```text
http://ВАШ_ВНЕШНИЙ_IP:5000
```
или по домену.

## Разрешение порта в Windows
Откройте PowerShell **от имени администратора**:
```powershell
New-NetFirewallRule -DisplayName "Public Lectory 5000" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
```

## Важные ограничения
- Пока ваш ПК выключен, сайт недоступен.
- Если провайдер использует серый IP, внешний доступ без VPN/туннеля/аренды сервера может не работать.
- SQLite подходит для учебного проекта и небольшого числа пользователей, но не для высокой нагрузки.
- Для настоящей публичной эксплуатации лучше использовать VPS.

## Альтернатива для быстрого внешнего доступа
Если не хочется настраивать роутер, можно временно использовать туннели вроде Cloudflare Tunnel или ngrok.

## Production на Linux/VPS
Если позже захотите вынести проект с ПК на сервер:
```bash
gunicorn wsgi:app
```

## Структура
- `app.py` — Flask-приложение
- `run_public.py` — запуск сервера на ПК через Waitress
- `wsgi.py` — точка входа для production
- `start_public_server.bat` — запуск на Windows
- `start_public_server.sh` — запуск на Linux/macOS
- `templates/` — HTML-шаблоны
- `static/css/` — стили
- `data/lectory.db` — SQLite база данных
